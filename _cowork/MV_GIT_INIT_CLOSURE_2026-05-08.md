## 6. Surprises and judgment calls

Things the brief didn't fully anticipate:

**FUSE-mount issues with `git init`.** Three separate problems hit in sequence:

  1. `mcp__cowork__allow_cowork_file_delete` was first rejected for `MediaVault/` (whole folder). I narrowed the request to `.git/` specifically and explained that no user files would be touched — only git's internal lock-file management. That was approved.
  2. After permission was granted, `git init` still produced a broken `.git/config`: 54 null bytes from bash's perspective via the FUSE mount, while the Read tool (host-side, bypassing FUSE) showed 3 lines of valid (but truncated) content. Classic FUSE Edit-truncation symptom from `weird-baby-museum/CLAUDE.md` quirk #1, but applied to git's atomic `.lock`-rename pattern instead of the Edit tool.
  3. `git init` also failed to create `.git/objects/` (and its `info/`, `pack/` subdirs). Without that, git reports "not a git repository."

  Workarounds: (a) used the Python rm-then-write pattern from CLAUDE.md to write `.git/config` correctly via FUSE; (b) `mkdir -p .git/objects/info .git/objects/pack` to fill the missing structure. After both, `git status` worked and the rest of the session ran without further git issues. Wrote the user/email/autocrlf config the same way (rm-then-write) instead of trusting `git config --local`.

**`Edit` and `Write` tools are blocked for `.git/` paths.** The host-side Write/Edit refused with "resolves to a protected location." Useful to know — `.git/` writes have to go through bash via FUSE.

**Layout drift between SPEC.md §11 and the live tree.** The brief's draft `.gitignore` named `catalogs/_assets/` and `catalogs/_thumbs/`, matching SPEC.md §11. The actual layout is `catalogs/<domain>/_thumbs/` (v0.4-style, e.g. `catalogs/hunter_root/_thumbs/`) plus `catalogs/vaulted/2026/04/` (v0.5-style for new artifacts). Both contain operator content; ignoring the whole `catalogs/` tree was cleaner than enumerating the sub-paths. SPEC.md §11 may want a sync to reality on a future pass — flagging it but not touching it (off-limits per session brief).
